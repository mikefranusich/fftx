template<int DIM, typename T>
struct array_t¶
#include <fftx3.hpp>

non-owning view into a contiugous array of data. This is a class that is foeshadowing a C++ class mdspan, a multi-dimensional extention to std::span

if fftx::tracing == true, then array_t::array_t(const box_t<DIM>& ) construction is a symbolic placeholder in a computational DAG that is translated into the code generator.

if fftx::tracing == false, then array_t::array_t(const box_t<DIM>&) will allocate a global_ptr sized to hold box_t::size elements of data.

template<typename T>
class global_ptr¶
#include <fftx3.hpp>

non-owning global ptr object. Can be used in place of upcxx::global_ptr

Most of the FFTX API assumes that user application code owns their primary data structures. This class encapsulates a user space raw data pointer for use within our transforms. The destructor does not delete the pointer. It is not reference-counting. It is not made to be unique. They can be copied moved, etc like a basic C struct.
